# ToyLink AI 项目交接文档 - 2026-07-06

## 当前状态

ToyLink AI 是一个 Flutter-first、Android-priority 的硬件桥接应用。当前产品主线是：

```text
Claude 原对话 -> 公网 Remote Bridge -> ToyLink App -> SafetyGuard -> ToyDevice -> BLE
```

核心原则仍然不变：

- ToyLink 是硬件桥，不是内置聊天替代品。
- UI 不能直接调用 BLE、不能构造协议字节。
- 所有控制必须经过 `McpToolRouter` / application use case / `SafetyGuard` / `ToyDevice`。
- Remote Bridge 当前应只对外开放 `get_status` 和 `stop_all`。
- `stop_all` 是最高优先级安全能力，App shell 已经提供全局急停条。

## 本次交接前完成的关键工作

- 全局 `AppShell` 急停条接入 `controlDeviceUseCaseProvider.stopAll()`。
- `ShellRoute` 将主要页面包进全局 shell，确保急停入口常驻。
- SOSEXY 写队列补了 `stopAll()` 抢占逻辑，避免被抢占命令的 Future 悬挂。
- Remote Bridge 默认 advertised tools 收敛为 `get_status,stop_all`。
- Bridge server 增加安全启动约束：
  - 非 loopback / 公网 bridge bind 必须配置 `BRIDGE_SHARED_TOKEN`。
  - `BRIDGE_DEBUG_TOKEN` 为空时 `/debug/enqueue` 返回 404，相当于禁用。
  - `/debug/enqueue` 配置 token 后仍要求鉴权。
- Docker Compose 要求设置 `BRIDGE_SHARED_TOKEN` 后才能暴露服务。
- `.gitignore` 已忽略根目录真实设备 XML/PNG 调试 dump、临时 txt、`docs/evidence/*.zip` 和未整理截图。

## 线上 Bridge 注意事项

2026-07-05 实测公网 Bridge：

- `GET http://47.95.242.29:8100/health` 返回 200。
- `GET http://47.95.242.29:8100/mcp/claude` 返回 200。
- `POST /mobile-bridge/session/start` 返回 connector 信息。

但线上部署当时仍存在两个问题：

- advertised tools 仍包含 `set_suck,set_vibe,set_ems,set_all`，说明线上未部署本地最新安全默认值。
- `/debug/enqueue` 当时无 token 也能排队任务。

本地代码已经修复这些默认行为。下一次部署前必须：

1. 重新构建 `bridge_server` 镜像。
2. 配置强随机 `BRIDGE_SHARED_TOKEN`。
3. 默认不要设置 `BRIDGE_DEBUG_TOKEN`；只有临时调试时设置，调试后清空。
4. 确认 `BRIDGE_TOOL_NAMES=get_status,stop_all`。
5. 部署后重新验证 `/mobile-bridge/session/start` 的 `toolNames`。
